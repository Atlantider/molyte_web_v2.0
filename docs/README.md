## 📝 文档规范

### 文档命名

- 使用英文命名
- 使用下划线分隔单词
- 使用大写字母（如：`QC_DESIGN.md`）
- 功能文档使用功能名前缀（如：`QC_`, `RDF_`, `ADMIN_`）

### 文档结构

每个文档应包含：

1. **标题** - 清晰的文档标题
2. **目录** - 章节导航（长文档）
3. **概述** - 简要说明
4. **详细内容** - 分章节说明
5. **示例** - 代码示例或使用示例
6. **参考** - 相关文档链接

### Markdown格式

- 使用标准Markdown语法
- 代码块指定语言（```python, ```bash等）
- 使用表格展示结构化数据
- 使用emoji增强可读性（适度）
